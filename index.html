HTML5 300: HTML5 Application Programming Interface (Apis)

Homework 1

Number Form

Create a Web page with an input field where the user enters numbers.

Each time the user enters a number, update the display to show how many numbers have been entered, the sum of the numbers, and the average of the numbers.

There should be a Reset button that allows the user to start over with a new sequence of numbers.

Your page should display an error message or otherwise prevent non-numeric input.

Optional features: display the standard deviation and/or median of the numbers entered

Homework 2

Object List

You will create a Web site that manages a simple in-memory database. The database will be a list (array) of objects. What the objects represent is up to you. They could be movies, contacts, a company's laptops, etc.

Create an HTML document with two sections (e.g., divs) which we will use as "pages". In the first section create a table with a heading whose columns are for the fields (properties) in your object database. Give it an empty tbody with an ID you will use in your JavaScript to populate the table. Make a button for adding new items.

In the second section, create a form with fields for the properties of your objects. Set the style of this section to "display:none" so it starts off hidden. Give both sections IDs so you can show and hide them in your JavaScript.

In your JavaScript, create an array variable to hold your data.

Write a function that creates a table row (tr) for each object in the array, with a cell (td) for each property to be displayed. Append each of these rows to the table body.

Write a handler for the new-item button. It should hide the table and show the form. It should set a handler for the form's submit button.

Write a handler for the submit button that creates an object using the form data and adds that object to the array. Then it should call the first function you wrote to recreate the table with the updated database and show that table, hiding the form.

Optional: On each row of the table, add buttons to edit and delete the object. This is a bit harder. To make this work, you should give each object a unique ID when you create it, and set that ID as an attribute (e.g. data-id) of the corresponding table row (or of each button), so you can determine which object to edit or delete. For editing, you can reuse the same form, but you'll have to modify the code to distinguish between adding a new object and editing an existing one.

Homework 3

Numbers and Strings

Create a Web page with a textarea input field where the user enters numbers or text.

Each time the user submits the input, check whether it is a number. If it is a number, update the display to show how many numbers have been entered, the sum of the numbers, and the average of the numbers. If it is a string, concatenate it onto all of the other strings that have been entered, and display the number of strings that have been entered and the accumulated concatenation.

There should be a Reset button that allows the user to start over with a new sequence of numbers.

Optional feature: Determine, update, and display the frequency of words entered in the strings. Hints: Use an object where the property keys (names) will be the words you find, and the property values will be the number of times each is encountered. Split a string into an array of words. For each word in the array, if the word is already a property name, increment its value. Otherwise set the value to 1. To display the frequencies, get the object's keys and make a row in a table for each one.
